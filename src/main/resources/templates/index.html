API Endpoints

POST /api/v1/images/compress

Compress image from a multipart form. Best for browser-based uploads.

Request: Multipart file with optional `quality` (0.0-1.0)

Response: Compressed image file (PNG)

POST /api/v1/images/compress-by-bytes

Compress image from raw bytes. Best for server-to-server integration.

Request: Raw image bytes in body, `Content-Type` header (e.g., `image/jpeg`), optional `filename` and `quality` parameters.

Response: Compressed image file (PNG)

GET /api/v1/images/health

Check the health status of the API.

Response: JSON object with status.
